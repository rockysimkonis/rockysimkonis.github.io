Big O Complexity and Notation

  • Rocky S.
  • Feb 21, 2016
  • Sharable links

The Big O notation is the language used to determine how fast an algorithm will run. Basically it is a method used to measure the efficiency of a particular problem. The measurement test with big O notation is testing the runtime as it grows relative to the input.

This can be broken down into a couple steps:

  1. Runtime growth
    • Because you can have multiple factors that can affect runtime such as processor speed and ram it is not so easy to make accurate predictions on algorithm speed. Therefore the measurement used is big O notation.
  2. Size of input
    • The runtime growth is expressed in (O(n))
  3. As input grows
    • The most important part is measuring how fast something runs as the the input grows. As n grows everything else becomes irrelevant.

Here is an example below where the runtime will grow as the input increases in size
def contains?(array, value)
array.each do |item|
return true if item == value
end
false
end
